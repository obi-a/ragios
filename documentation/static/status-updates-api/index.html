Status Updates API

Status Updates
POST
To add a status update to the system

POST /status_updates HTTP/1.0
Content-Type: application/json
 
 { 
   "every" : "24h",
   "contact" : "admin@ateam.com",
   "via" : "gmail"
 }

This will add a status update to the system that will be sent out every 24 hours to the email address admin@ateam.com via gmail.

GET
To get all status updates in the system

GET /status_updates

A status update in JSON format could look like this

 {  
   "every" : "1w",
   "contact" : "admin@ateam.com",
   "via" : "gmail",
   "tag" : "admin" 
 }

You can get status updates using their key/value pair values.

Example to get all status updates scheduled to run every week,

GET /status_updates/every/1w

To get all status updates with contact admin@ateam.com

GET /status_updates/contact/admin@ateam.com

To get all status updates that send notifications via gmail

GET /status_updates/via/gmail

To get all status updates tagged admin

GET /status_updates/tag/admin

Syntax

GET /status_updates/:key/:value

The JSON format for an active status update may look like this:

{
   "_id": "df96c45d-d00c-4a9c-b410-0f169357ca12",
   "_rev": "2-58d0708e083c2c521b41fdeed3767303",
   "every": "1m",
   "contact": "test@mail.com",
   "via": "gmail",
   "tag": "save_test",
   "state": "stopped"
}

All status upates in the system have a unique _id value. This unique _id is generated by Ragios. To get a status update by its unique id
Syntax:

 GET /status_updates/:id

Example

 GET /status_updates/df96c45d-d00c-4a9c-b410-0f169357ca12

DELETE
To get a status update, we pass it’s tag to a status updates DELETE request.
Syntax

DELETE /status_updates/:tag

To delete all status updates tagged admin

DELETE /status_updates/admin

If the status update with the specified tag is active, it will be stopped and then deleted.

PUT
Edit a status update
To edit a status update we use it’s unique id. Example to change the notification email of a status update;

 PUT /status_updates/some_status_update_id HTTP/1.0
 Content-Type: application/json
 
 { "contact" : "john@ateam.com"}

This will change the contact for the status update with the specified unique id to john@ateam.com

To edit the time interval used by a status update to send out status reports,

 PUT /status_updates/some_status_update_id HTTP/1.0
 Content-Type: application/json
 
 { "every" : "2w"}

This change the time interval for the status update with the specified id to 2 weeks.

To change the notification email, tag and the time interval in one PUT request

 PUT /status_updates/some_status_update_id HTTP/1.0
 Content-Type: application/json
 
{
   "every": "24h",
   "contact": "james@ateam.com",
   "tag": "james"
}

Stop or restart a status update
To stop a running status update
syntax

PUT /status_updates/:tag/state/stopped

Example

PUT /status_updates/admin/state/stopped

This will stop all status updates with the specified tag.

To restart a stopped monitor
syntax:

/status_updates/:tag/state/active
/status_updates/admin/state/active

This will restart all status updates with the specified tag.
